# pnpm + monorepo + vue

# monorepos 介绍

Monorepo 指的是单一代码仓库（Monolithic Repository），这种方式是将多个相关的项目或模块集中在一个代码仓库中进行管理和开发。在 Monorepo 中，所有的代码都存在于同一个版本控制系统中，这样可以使得开发人员更加方便地进行跨模块和跨项目的协作和统一管理。Monorepo 通常会使用工具和框架来帮助开发人员管理依赖关系、构建和部署等方面的工作。Monorepo 的优点是可以减少代码复制、降低代码维护成本、提高代码复用率、简化部署流程等。

# monorepos 优点

  1. 提高协作效率：将多个相关的项目或模块集中在一个代码仓库中进行管理和开发，可以减少代码复制，降低代码维护成本，提高协作效率。

  2. 简化依赖关系管理：Monorepo 可以更方便地管理依赖关系，避免不同项目或模块之间的依赖关系冲突和版本不一致的问题。

  3. 提高代码复用率：Monorepo 可以提高代码复用率，因为不同项目或模块之间可以更方便地共享代码。

  4. 简化部署流程：Monorepo 可以简化部署流程，因为所有的代码都存在于同一个版本控制系统中，可以更方便地进行构建和部署。

  5. 提高代码可维护性：Monorepo 可以提高代码可维护性，因为所有的代码都存在于同一个版本控制系统中，可以更方便地进行代码管理和版本控制。

# monorepos 缺点

  1. 大型代码库的维护难度：Monorepo 适用于大型项目，但是当代码库变得非常庞大时，代码库的维护和管理可能会变得非常困难。

  2. 构建和部署时间变长：由于 Monorepo 中存在多个项目或模块，构建和部署时间可能会变长。

  3. 代码库的耦合性增加：由于 Monorepo 中所有项目或模块都存在于同一个代码库中，可能会导致不同项目或模块之间的耦合性增加，代码库的复杂度也会增加。

  4. 需要更高的自动化水平：Monorepo 需要更高的自动化水平，包括代码检查、自动化测试等，以确保代码库的稳定性和可维护性。

  5. 可能需要更多的硬件资源：由于 Monorepo 中存在多个项目或模块，可能需要更多的硬件资源来支持代码库的构建和部署。

# pnpm

由于 pnpm 开箱就对 monorepo 的支持，所以本次使用 pnpm 搭建项目。

# 项目环境搭建

  1. 首先安装 pnpm
 npm i pnpm -g
  1. 新建文件夹
    # 这里随意，只需要在电脑上新建一个文件夹就好了
    mkdir pnpm-monorepo-vue
  1. 初始化项目
    pnpm init
  • 此时会生成一个 package.json 文件
{
  "name": "pnpm-monorepo-vue",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],
  "author": "",
  "license": "ISC"
}
  1. 创建 packages 文件夹，待会我们会把包放到该文件夹中管理
    mkdir packages
  1. 创建 pnpm-workspace.yaml 文件
  • pnpm-workspace.yaml 是一个配置文件，用于配置 pnpm 工作区。它可以包含以下信息：
  • 工作区的根目录
  • 工作区中的包，以及它们的依赖关系
  • 工作区中的包的版本
  • 工作区中的包的位置
touch pnpm-workspace.yaml

把一下内容粘贴到 pnpm-workspace.yaml 文件中，它的意思是说明 packages/文件下的所有文件都是该工作区中的包。

packages: -"packages/*";
  1. 进入 packages 新建两个项目.一个是 Vue，一个是 Common。
     cd packages/
        vue create vue
        cd ..
     mkdir Common
        pnpm init # 初始化Common项目

此时两个项目都创建完毕了

  1. 在 Common 中创建 index.js，并且 export 一个方法给 vue 项目使用
    touch index.js

index.js

export function say() {
  console.log("Common");
}
  1. 修改 Common 文件夹中 package.json 的 name 字段，name 以项目名来命名
{
  "name": "@Common/util",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],
  "author": "",
  "license": "ISC"
}

8.在 vue 项目中使用 common 包的 say 方法，需要在 vue 项目中安装 common 的依赖，我们可以通过 pnpm 来安装

cd packages/vue
# 这里的意思是，安装@Common/util指定vue项目。 pnpm --filter的意思是指定哪个项目
pnpm install @Common/util --filter vue
